# Contributing

## Updating the dataset

The JSON files in `data/` are generated by the Cocoajam SRD parser — **do not edit them by hand**.
To update the dataset:

1. Run the parser with the `--publish-to` flag pointed at this repo:

   ```bash
   cd cocoajam/scripts/srd-parse
   python parse.py --publish-to ../../srd-5.2.1/repo
   ```

   This regenerates both the Encounter Manager's in-repo JSON *and* the public repo's `data/` folder in one command.

2. Review the diff in `data/`.

3. Bump the version in `CHANGELOG.md` and `mcp/package.json`.

4. Rebuild the MCP server:

   ```bash
   cd mcp
   npm run prepare   # copies updated data + rebuilds
   npm test          # must pass
   ```

5. Commit, tag, and publish:

   ```bash
   git add -A
   git commit -m "chore: update SRD dataset YYYY-MM-DD"
   git tag vX.Y.Z
   git push origin main --tags
   cd mcp && npm publish
   ```

## Reporting data issues

If a monster or condition value looks wrong, open an issue with:
- The monster/condition name
- The field that looks incorrect
- What the SRD PDF says vs. what we have
- Whether Open5e also has the wrong value

Known discrepancies are documented in [data/expected-mismatches.md](data/expected-mismatches.md).

## Schema changes

Schema files in `data/schema/` are hand-written. If the parser adds new fields, update the schema
and add a test case to `mcp/src/test/tools.test.ts`. Schema changes require a minor version bump.
