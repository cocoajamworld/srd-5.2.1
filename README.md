## What's in this repo

| Path | What |
|------|------|
| `data/monsters.json` | 322 monsters — full stat blocks in structured JSON |
| `data/conditions.json` | 14 conditions with markdown-formatted rules text |
| `data/schema/monster.schema.json` | JSON Schema (draft-07) for `monsters.json` |
| `data/schema/condition.schema.json` | JSON Schema (draft-07) for `conditions.json` |
| `data/expected-mismatches.md` | Known differences from the Open5e cross-reference |
| `mcp/` | stdio MCP server — `lookup_monster`, `lookup_condition`, `search_monsters`, `license` |

This is SRD **5.2.1** (2024 rules), not the 2014 SRD. The data is sourced from the Open5e API
(`document__slug=wotc-srd`) which publishes the same CC-BY 4.0 licensed content in structured form.

## Install the MCP server

### Claude Desktop

Add to your `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "srd-521": {
      "command": "npx",
      "args": ["-y", "@cocoajamworld/srd-5.2.1-mcp"]
    }
  }
}
```

### Claude Code (CLI)

```bash
claude mcp add srd-521 -- npx -y @cocoajamworld/srd-5.2.1-mcp
```

> **Note:** MCP server names can't contain dots — use `srd-521` (or any name you like).

### Run directly

```bash
npx @cocoajamworld/srd-5.2.1-mcp
```

### Available tools

| Tool | Description |
|------|-------------|
| `lookup_monster` | Full stat block for a named monster (`{ name: "goblin" }`) |
| `lookup_condition` | Condition rules for a named condition (`{ name: "charmed" }`) |
| `search_monsters` | Filter monsters by CR, type, and/or size (`{ cr: "1/4", type: "humanoid" }`) |
| `license` | Returns the full CC-BY 4.0 attribution block |

Every response includes a `_license` field with the CC-BY 4.0 attribution string.

## Use the data directly

The JSON files are plain arrays wrapped in an object with a top-level `_license` field:

```bash
# All goblin fields
jq '.monsters[] | select(.name == "Goblin")' data/monsters.json

# All CR 1/4 humanoids
jq '[.monsters[] | select(.cr == "1/4" and .type == "humanoid") | .name]' data/monsters.json

# All conditions
jq '[.conditions[].name]' data/conditions.json
```

Validate against the schemas:

```bash
npx ajv-cli validate -s data/schema/monster.schema.json -d data/monsters.json
npx ajv-cli validate -s data/schema/condition.schema.json -d data/conditions.json
```
